CLI Flags

Session Management

--continue, -c

Load the most recent conversation in the current directory

--resume, -r <session>

Resume a specific session by ID or name, or show an interactive picker

--fork-session

When resuming, create a new session ID instead of reusing the original

--session-id <uuid>

Use a specific session ID for the conversation (must be a valid UUID)

--name, -n <name>

Set a display name for the session, shown in /resume and terminal title

--from-pr <number_or_url>

Resume sessions linked to a specific GitHub PR

Input / Output & Print Mode

--print, -p

Print response without interactive mode

--output-format <format>

Output format for print mode: text, json, stream-json

--input-format <format>

Input format for print mode: text, stream-json

--include-partial-messages

Include partial streaming events in output (requires --print and --output-format stream-json)

--include-hook-events

Include all hook lifecycle events in the output stream (requires --output-format stream-json)

--replay-user-messages

Re-emit user messages from stdin back on stdout for acknowledgment (requires stream-json I/O)

--json-schema '<schema>'

Get validated JSON output matching a JSON Schema after agent completes workflow (print mode only)

Model Configuration

--model <model>

Set the model for the current session (alias: sonnet, opus, or full model name)

--effort <level>

Set effort level: low, medium, high, max

--fallback-model <model>

Enable automatic fallback to specified model when default is overloaded (print mode only)

System Prompt Customization

--system-prompt "<text>"

Replace the entire system prompt with custom text

--system-prompt-file <path>

Load system prompt from a file, replacing the default prompt

--append-system-prompt "<text>"

Append custom text to the end of the default system prompt

--append-system-prompt-file <path>

Load additional system prompt text from a file and append to the default prompt

File & Directory Access

--add-dir <path> [<path> ...]

Add additional working directories for Claude to read and edit files

--plugin-dir <path>

Load plugins from a directory for this session only (repeat for multiple)

Permissions & Security

--permission-mode <mode>

Begin in a specified permission mode: default, acceptEdits, plan, auto, dontAsk, bypassPermissions

--dangerously-skip-permissions

Skip permission prompts (equivalent to --permission-mode bypassPermissions)

--allow-dangerously-skip-permissions

Add bypassPermissions to the Shift+Tab mode cycle without starting in it

--enable-auto-mode

Unlock auto mode in the Shift+Tab cycle (requires Team/Enterprise/API plan and Sonnet/Opus 4.6)

--allowedTools "<rules>"

Tools that execute without prompting for permission

--disallowedTools "<rules>"

Tools that are removed from the model's context and cannot be used

--tools "<tools>"

Restrict which built-in tools Claude can use ("" to disable all, "default" for all)

--permission-prompt-tool <mcp_tool>

Specify an MCP tool to handle permission prompts in non-interactive mode

Configuration

--settings <path_or_json>

Path to a settings JSON file or a JSON string to load additional settings

--setting-sources <sources>

Comma-separated list of setting sources to load: user, project, local

Agent & Subagent

--agent <agent>

Specify an agent for the current session (overrides the agent setting)

--agents '<json>'

Define custom subagents dynamically via JSON

MCP (Model Context Protocol)

--mcp-config <files_or_strings>

Load MCP servers from JSON files or strings (space-separated)

--strict-mcp-config

Only use MCP servers from --mcp-config, ignoring all other MCP configurations

Browser & IDE Integration

--chrome

Enable Chrome browser integration for web automation and testing

--no-chrome

Disable Chrome browser integration for this session

--ide

Automatically connect to IDE on startup if exactly one valid IDE is available

Remote & Web Sessions

--remote "<task>"

Create a new web session on claude.ai with the provided task description

--remote-control, --rc

Start interactive session with Remote Control enabled (controllable from claude.ai)

--teleport

Resume a web session in your local terminal

Git Worktrees

--worktree, -w [<name>]

Start Claude in an isolated git worktree at <repo>/.claude/worktrees/<name>

--tmux

Create a tmux session for the worktree (requires --worktree); uses iTerm2 native panes when available

Channels (Research Preview)

--channels '<plugins>'

MCP servers whose channel notifications Claude should listen for (space-separated)

--dangerously-load-development-channels

Enable channels not on the approved allowlist for local development

Advanced Options

--bare

Minimal mode: skip auto-discovery of hooks, skills, plugins, MCP servers, auto memory, and CLAUDE.md

--init

Run initialization hooks and start interactive mode

--init-only

Run initialization hooks and exit (no interactive session)

--maintenance

Run maintenance hooks and start interactive mode

--disable-slash-commands

Disable all skills and commands for this session

--no-session-persistence

Disable session persistence so sessions are not saved to disk (print mode only)

--max-turns <number>

Limit the number of agentic turns (print mode only)

--max-budget-usd <amount>

Maximum dollar amount to spend on API calls before stopping (print mode only)

--betas <headers>

Beta headers to include in API requests (API key users only)

--teammate-mode <mode>

Set how agent team teammates display: auto, in-process, or tmux

Debugging & Troubleshooting

--debug [<categories>]

Enable debug mode with optional category filtering (e.g. "api,hooks" or "!statsig,!file")

--debug-file <path>

Write debug logs to a specific file path (implicitly enables debug mode)

--verbose

Enable verbose logging, shows full turn-by-turn output

Version & Help

--version, -v

Output the version number
